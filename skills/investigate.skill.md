# Investigation Skill

## Goal
Understand system behavior, structure, and relationships deeply.

---

## Context

Use:

- `.github/utility/linking/change_log/` → existing implementations
- `.github/utility/linking/planning_log/` → intended flow (reference only)

---

## Steps

1. Identify Scope
- Locate relevant files, modules, and components
- Define target area clearly

2. Map Structure
- Identify key functions and modules
- understand their role in system

3. Trace Relationships
- map dependencies between components
- identify data and control flow

4. Analyze Usage
- find where and how components are used
- check interactions across modules

5. Identify Behavior
- understand expected vs actual behavior
- detect unusual or complex patterns

6. Expand Context
- look into related modules
- identify potential impact areas

---

## Rules

- Focus on understanding, not implementation
- Do NOT modify:
  `.github/utility/linking/planning_log/`
- Keep scope controlled (avoid full repo scan unless needed)
- Use change_log as execution reference
- Do not generate code unless strictly necessary for exploration
