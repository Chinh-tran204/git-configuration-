# Investigation Skill

## Goal
Understand system behavior, structure, and relationships deeply

## Context
- Read: planning_log (what to do, read-only)
- Read: codebase (implementation details)
- Scope: relevant files/modules only

## Steps
1. **Define Scope** → locate relevant files, modules, and components; define target area
2. **Map Structure** → identify key functions and modules; understand their role
3. **Trace Relationships** → map dependencies between components; identify data and control flow
4. **Analyze Usage** → find where components are used; check cross-module interactions
5. **Identify Behavior** → understand expected vs actual behavior; detect patterns
6. **Expand Context** → look into related modules; identify impact areas

## File-Aware Rules
- Focus on relevant scope only (avoid full-repo scanning)
- Track file paths and line numbers for references
- Document relationships clearly for future tracing
